# ОТЧЕТ ПО ЛАБОРАТОРНОЙ РАБОТЕ №2: SECURITY PERMISSIONS

Данная документация описывает архитектуру, требования и порядок эксплуатации мобильного приложения "Vector Lab", разработанного в рамках курса по информационной безопасности мобильных приложений.

---

## 1. ОПИСАНИЕ ПРОЕКТА
Цель работы — реализация системы управления разрешениями в реальном времени (Runtime Permissions) для ОС Android. Приложение демонстрирует разграничение доступа к системным ресурсам и предотвращает утечку конфиденциальной информации через несанкционированный доступ к API устройства.

### Реализованные цели:
* **Основное задание**: Приложение не запрашивает права при установке и отображает актуальный статус разрешений в UI.
* **Дополнительные баллы**: 
    1. Реализовано 3 различных типа разрешений: CAMERA, CONTACTS, LOCATION.
    2. Разработан внешний скрипт на Python для отзыва прав через ADB.
    3. Добавлен интерфейс для перехода в системные настройки для ручного управления доступом.

---

## 2. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ
* **ОС мобильного устройства**: Android API level 23 (Marshmallow) и выше.
* **ОС хост-машины**: Windows/Linux/macOS (для запуска скрипта отзыва).
* **Среда разработки**: Android Studio.
* **Инструментарий**: ADB (Android Debug Bridge), Python 3.x.

---

## 3. АРХИТЕКТУРА И РЕАЛИЗАЦИЯ

### Мобильное приложение (Java/Android)
Приложение построено на базе активности ```MainActivity```, которая контролирует жизненный цикл разрешений:
1. **Проверка**: При каждом запуске или возврате в приложение (метод ```onResume```) происходит опрос системы о текущем статусе разрешений через ``ContextCompat.checkSelfPermission``.
2. **Отображение**: UI-компоненты динамически меняют текст и цвет (Зеленый - GRANTED, Красный - DENIED).
3. **Запрос**: При нажатии "REQUEST ALL" вызывается системный диалог ```requestPermissions```.

### Скрипт автоматизации (Python)
Для выполнения требований по отзыву прав со стороны хост-машины используется скрипт ```revoke_permissions.py```:
* Команда ```adb shell pm reset-permissions``` принудительно сбрасывает все ранее выданные runtime-привилегии для пакета приложения.
* Это позволяет имитировать сценарий компрометации или административного ограничения прав без участия пользователя.

## 4. ИНСТРУКЦИЯ ПО ИСПОЛЬЗОВАНИЮ

### Сценарий проверки разрешений:
1. Установите приложение на эмулятор или физическое устройство.
2. Убедитесь, что изначально все индикаторы находятся в статусе **DENIED**.
3. Нажмите кнопку **"REQUEST ALL"** и выдайте разрешения в системных диалогах. Статусы изменятся на **GRANTED**.
4. Нажмите **"OPEN APP SETTINGS"**, чтобы вручную отозвать одно из прав в настройках Android, затем вернитесь в приложение — статус обновится автоматически.

### Работа со скриптом отзыва (ADB):
1. Подключите устройство к ПК и убедитесь, что оно отображается в команде ```adb devices```.
2. Запустите скрипт: ```python revoke_permissions.py```.
3. После завершения работы скрипта откройте приложение — все права будут сброшены.
